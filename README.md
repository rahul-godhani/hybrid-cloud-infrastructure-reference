## Overview
This repository presents a reference hybrid cloud infrastructure design
covering AWS and Microsoft Azure, with a focus on operational reliability,
security, and maintainability.

## Why Hybrid Cloud
Many organizations operate across on-premise, AWS, and Azure due to
business, compliance, or legacy constraints. This design reflects a
practical, operations-focused hybrid approach.

## Design Principles
- Infrastructure as Code
- Least-privilege security
- High availability by design
- Operational simplicity

## What This Repository Covers
- AWS and Azure environment structure
- Networking and security concepts
- Reliability and operational considerations

## What This Repository Does Not Cover
- Application-level design
- CI/CD pipelines
- Vendor-specific optimizations

## How to Use This Repository
This repository is intended as a reference and discussion artifact,
not a production-ready deployment.
